Analise do Projeto Rota do Boto

Frontend React/Vite para compra de passagens fluviais com Firebase Auth, Firestore, carrinho local e backend de reservas.

Resumo

Build: npm run build passou.

Lint: npm run lint falhou com 24 erros e 1 aviso.

Os riscos principais estao em autenticacao, rotas dependentes de location.state, datas inconsistentes e URLs de backend locais.

Logica Principal

  1. Home em / mostra busca e cards.
  2. TravelSearchCard.jsx aceita apenas Manaus para Parintins em maio de 2026.
  3. Busca navega para /resultados usando location.state.
  4. resultados.jsx mostra lanchas hardcoded e envia para /pedido.
  5. pedido.jsx consulta assentos e chama http://127.0.0.1:8000/reserve.
  6. Carrinho guarda a reserva temporaria e checkout chama http://127.0.0.1:8000/confirm.

Recomendacoes

  1. Adicionar loading real ao AuthContext.
  2. Proteger /pedido, /carrinho e /checkout.
  3. Tratar falta de location.state em /resultados e /pedido.
  4. Unificar datas e dados de viagem.
  5. Conectar Historico no perfil ou remover o arquivo.
  6. Usar VITE_API_URL para a URL do backend.
