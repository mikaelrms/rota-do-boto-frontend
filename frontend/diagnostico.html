Diagnóstico — Frontend & Backend

Resumo rápido das áreas mais prováveis de falha e ações recomendadas. Verifique cada item no ambiente de desenvolvimento.

Frontend (frontend/rota-do-boto/frontend)

  • index.html carrega /src/main.jsx e tem <div id="root"> — OK; se o app não renderizar, verificar console do navegador e build do Vite.
  • Firebase: src/firebaseConfig.js contém a configuração (apiKey etc.). Chave pública do cliente é normal, mas: restrinja origens no console Firebase e prefira usar variáveis de ambiente para VITE_* (p.ex. VITE_FIREBASE_PROJECT ou VITE_API_URL).
  • API URL: código usa import.meta.env.VITE_API_URL || "http://127.0.0.1:8000". Se a UI não conseguir falar com a API, criar um arquivo .env na raiz do frontend com VITE_API_URL=http://seu-backend:8000 e reiniciar o dev server (vite).
  • Verificar compatibilidade de versões (react 19, react-router-dom v7 e uso de createBrowserRouter) — atualizar dependências se houver erro de API.
  • Tailwind/vite plugin está nas dependências. Se o CSS faltar, checar tailwind.config.js e se o plugin foi registrado no vite.config.js.
  • Possíveis erros runtime: imports com caminho incorreto (extensões .jsx vs .js), erro em AuthContext (onAuthStateChanged) se o objeto auth não for inicializado; ver console do navegador e network.

Backend (backend/rota-do-boto-backend/backend)

  • Projeto em Python + Firestore (FastAPI). Requisitos em requirements.txt.
  • Import paths: src/main.py faz from src import firebase_service as fb e importa tasks.expiration_worker. Se uvicorn for executado com o cwd incorreto, imports relativos podem falhar. Recomendações:
    • Iniciar o servidor com o módulo do package: cd backend/rota-do-boto-backend/backend && uvicorn src.main:app --reload --host 0.0.0.0 --port 8000 ou ajustar PYTHONPATH para incluir a raiz onde src é pacote.
    • Alternativa: usar imports relativos dentro de src (ex: from . import firebase_service) caso rode como package (python -m src.main).
  • Credenciais Firebase: src/firebase_config.py usa os.getenv('FIREBASE_KEY_PATH','firebasekey.json'). Problemas comuns:
    • Arquivo firebasekey.json não encontrado quando o cwd não é o diretório esperado — usar caminho absoluto via variável de ambiente FIREBASE_KEY_PATH.
  • Worker de expiração: roda em thread (tasks/expiration_worker.py) e chama src.firebase_service.cancel_order. Se o serviço iniciar e sair ou travar, checar logs no terminal (prints/tracebacks) e permissões de leitura/escrita no Firestore.
  • Possíveis erros de runtime no backend:
    • Erros de import (ModuleNotFoundError) por execução a partir de pasta errada.
    • Erro ao carregar credenciais do Firebase (ValueError/IOError) se a JSON estiver inválida ou o caminho incorreto.
    • Problemas de versão das bibliotecas Google/FireStore — instalar requisitos em venv limpo e testar.

Problemas de integração comuns

  1. Frontend não encontra a API — verificar VITE_API_URL, CORS (backend permite "*"), e se o backend está de fato escutando na porta esperada.
  2. Auth não funciona — verificar console do browser para erros do SDK Firebase; confirmar que auth foi inicializado antes do onAuthStateChanged.
  3. Tasks em background morrem na inicialização do processo (erros silenciosos) — rodar backend em terminal e observar logs.

Passos sugeridos para validar localmente

  1. Backend: criar e ativar venv, instalar dependências: python -m venv .venv && .\.venv\Scripts\Activate.ps1 (Windows PowerShell) e pip install -r requirements.txt.
  2. Definir variável de ambiente do Firebase: setx FIREBASE_KEY_PATH "C:\caminho\para\firebasekey.json" ou export temporário no shell.
  3. Iniciar backend (a partir de backend/.../backend): uvicorn src.main:app --reload --host 0.0.0.0 --port 8000.
  4. Frontend: criar .env na raiz do frontend com VITE_API_URL=http://127.0.0.1:8000, instalar dependências npm install e iniciar npm run dev. Ver console do Vite para erros.
  5. Verificar network tab e console no navegador para endpoints /trips/.../seats, /reserve, /confirm, /cancel e autenticação Firebase.

Recomendações rápidas

  • Usar variáveis VITE_* para URLs/flags do frontend.
  • Setar FIREBASE_KEY_PATH absoluto e executar backend a partir do diretório correto.
  • Preferir imports relativos dentro do package Python (from .) se o projeto for sempre executado como pacote.
  • Revisar permissões do Firestore e regras de segurança para produção.

Se desejar, rodar checagens automáticas (npm run build/test, iniciar uvicorn) e eu analiso os logs de erro resultantes — posso ajudar a aplicar correções específicas.
